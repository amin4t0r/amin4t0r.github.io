Test d'Intrusion Black Box — ESDown

Pentest boîte noire sur l'infrastructure d'une entreprise pharmaceutique fictive.

🎯 Objectifs

  • Compromettre un service exposé depuis l'IP publique
  • Effectuer une latéralisation vers le réseau interne (192.168.11.0/24)
  • Accéder aux recettes médicales et au code source métier

🗺️ Chemin d'attaque

  • Scan Nmap → port 80 ouvert, fuzzing Gobuster → répertoire /dev/
  • Injection SQL sur catalogue.html → dump complet de la base webapp
  • Exécution de commandes via SQLMap (os-shell) → accès au serveur
  • Identifiants MySQL en clair dans user.php → webshell PHP uploadé
  • Pivot réseau via Ligolo-ng vers le réseau interne
  • Découverte GitLab v13.9.5 → RCE via CVE-2021-22205
  • Exfiltration des recettes médicales depuis le dépôt Git interne

🐛 Vulnérabilités clés (14 identifiées)

  • Injection SQL critique (CVSS 9.8) — paramètre POST non filtré
  • RCE GitLab CVE-2021-22205 (CVSS 9.9) — version obsolète
  • Webshell PHP uploadé via /upload/ (CVSS 9.6) — aucun filtrage
  • Credentials en clair dans user.php et backup.sql
  • Pivot réseau possible vers le réseau interne

🔍 Audit SSI

Audit SSI — Annuaire d'authentification ESDown

Évaluation de la sécurité de l'Active Directory selon le guide d'hygiène ANSSI et PingCastle.

🎯 Contexte et objectifs

  • Évaluer la sécurité de l'annuaire d'authentification (Active Directory), composant central du SI
  • Identifier les vulnérabilités et mauvaises pratiques compromettant l'intégrité, la confidentialité et la disponibilité
  • Référentiels utilisés : Guide d'hygiène informatique ANSSI (volets Réseau et Administration) et rapport PingCastle
  • Méthodologie PDCA : Plan (périmètre, objectifs), Do (collecte technique), Check (analyse des écarts), Act (recommandations)

⚠️ 14 écarts identifiés

  • Absence de segmentation réseau bureautique (192.168.150.0/22)
  • Port RDP 3389 exposé directement sur internet
  • Aucun contrôle d'accès réseau (NAC) sur les prises RJ45
  • Contrôle d'accès physique insuffisant : baie ouverte, badge sans supervision
  • Algorithme DES vulnérable toujours activé sur l'annuaire
  • Protocole SMBv1 activé sur plusieurs contrôleurs de domaine
  • Politique de mot de passe faible, mot de passe krbtgt inchangé depuis 2 673 jours
  • Comptes à privilèges non membres du groupe "Protected Users", pas de revue périodique
  • LAPS non déployé : 856 comptes avec mots de passe locaux statiques

📈 Résultats

  • Score de maturité initial : 2/4 — Domain Risk Level PingCastle : 100/100
  • Plan de remédiation avec 14 mesures correctives priorisées (court/moyen terme)
  • Objectif visé après implémentation : 3,5 à 4/4 — Domain Risk Level cible : 0/100
